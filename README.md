# Proxmox Backup & Recovery

A two-part backup strategy for a single-node Proxmox VE host:

1. **Host configuration** — captured as Ansible vars via a harvester script, so the host
   can be rebuilt from scratch using the `lae.proxmox` Ansible role.
2. **VMs & containers** — backed up via `vzdump` with GFS rotation to a NAS.

## Quick Install

Run this on your Proxmox host as root — the installer will ask for your NAS details,
backup schedule, and retention settings interactively:

```bash
curl -sSL https://raw.githubusercontent.com/MarkLFT/proxmox-backup/master/install.sh | bash
```

Or download and inspect first:

```bash
curl -sSL -o install.sh https://raw.githubusercontent.com/MarkLFT/proxmox-backup/master/install.sh
less install.sh
bash install.sh
```

## Architecture

```
proxmox-backup/
├── install.sh                   # One-line curl installer
├── README.md
├── harvest-proxmox-config.sh    # Runs on PVE host, generates Ansible vars
├── proxmox-backup.sh            # Main backup script (harvester + vzdump + GFS)
├── proxmox-backup.conf          # Configuration file
├── install-cron.sh              # Sets up the cron job
│
├── ansible/                     # Ansible project for host recovery
│   ├── requirements.yml         # Galaxy dependencies (lae.proxmox, community.proxmox)
│   ├── inventory/
│   │   └── hosts.yml            # Inventory (edit with your host IP)
│   ├── host_vars/
│   │   └── pve.yml              # ← Generated by harvest script
│   ├── playbooks/
│   │   ├── recover-host.yml     # Full host rebuild playbook
│   │   └── restore-vms.yml     # VM restore playbook
│   └── roles/
│       └── proxmox-extras/      # Custom role for things lae.proxmox doesn't cover
│           └── tasks/
│               └── main.yml
│
└── recover-vms.sh               # Quick VM restore helper script
```

## Manual Setup

If you prefer not to use the curl installer:

### 1. Install on the Proxmox host

```bash
# Clone the repo
git clone https://github.com/MarkLFT/proxmox-backup.git /opt/proxmox-backup

# Edit config
vi /opt/proxmox-backup/proxmox-backup.conf

# Mount your NAS
mount -t nfs nas.local:/backups /mnt/nas-backup

# Install the cron job
bash /opt/proxmox-backup/install-cron.sh
```

### 2. First run (generates Ansible config + first backup)

```bash
bash /opt/proxmox-backup/proxmox-backup.sh
```

### 3. Recovery — rebuild host from scratch

```bash
# On your workstation with Ansible installed:
cd proxmox-backup/ansible
ansible-galaxy install -r requirements.yml
ansible-playbook playbooks/recover-host.yml -i inventory/hosts.yml
```

### 4. Recovery — restore VMs

```bash
# On the rebuilt host:
bash /opt/proxmox-backup/recover-vms.sh /mnt/nas-backup/proxmox/daily/2026-03-27
```

## GFS Rotation

| Tier    | Kept | Trigger              |
|---------|------|----------------------|
| Daily   | 7    | Every run            |
| Weekly  | 4    | Sundays              |
| Monthly | 6    | 1st of month         |

A daily backup that falls on a Sunday is stored as weekly instead. A backup on the 1st
is stored as monthly. This means each day produces exactly one backup in one tier.

## Dependencies

- **On the Proxmox host**: `bash`, `vzdump`, `qm`, `pct` (all included with PVE)
- **On the Ansible controller**: `ansible-core >= 2.15`, Python 3.10+
