# Proxmox Backup & Recovery

## Project Overview

A backup and disaster recovery solution for a single-node Proxmox VE host. Backs up host configuration (as Ansible vars), host config files, and all VMs/containers via vzdump with GFS (Grandfather-Father-Son) rotation to a NAS.

## Key Scripts

- `proxmox-backup.sh` — Main backup orchestrator (runs daily via cron)
- `install.sh` — Interactive curl-based installer
- `install-cron.sh` — Cron job installer/manager
- `harvest-proxmox-config.sh` — Generates Ansible host vars from running host config
- `recover-vms.sh` — VM/container restore helper
- `proxmox-backup.conf` — Configuration file (generated by installer)

## Architecture

- **GFS rotation**: daily/weekly/monthly tiers determined by `determine_gfs_tier()` based on date
  - 1st of month = monthly, configured weekday = weekly, everything else = daily
- **Dual backup**: Host config (Ansible vars + tar archive) + VM disk images (vzdump)
- **Storage modes**: Named Proxmox storage (`VZDUMP_STORAGE`) or raw dumpdir on NAS
- **Backup notes**: Automatically set via `--notes-template "{{guestname}} - ${tier} backup"`
- **Notifications**: Email via Proxmox's root@pam, configurable level (failure/always/none)

## Installer Validation Rules

- GFS retention values must be positive integers: daily (1-365), weekly (1-52), monthly (1-24)
- All three retention tiers require at least 1 — Proxmox UI warns when any tier is 0
- Cron hour: 0-23, minute: 0-59
- Uses `prompt_number` for validated numeric input, `prompt_choice` for enum selections

## Proxmox vzdump Notes

- `--notes-template` supports `{{guestname}}` placeholder for VM/CT name
- Notes are visible in the Proxmox UI backup content list
- `VZDUMP_EXTRA_ARGS` in conf can override notes if a custom `--notes-template` is provided (last one wins)

## Proxmox Prune/Retention Rules

- `pvesm set <storage> --prune-backups` accepts: keep-last, keep-hourly, keep-daily, keep-weekly, keep-monthly, keep-yearly (all integers >= 0)
- `keep-all=1` is mutually exclusive with other keep-* options
- If all keep-* values are 0, Proxmox defaults to keep-all (no pruning)
- No dependency rules between tiers, but the UI warns when shorter tiers are set without longer ones
- Processing order: keep-last, keep-hourly, keep-daily, keep-weekly, keep-monthly, keep-yearly (cumulative)

## Testing

Scripts run on Proxmox VE hosts as root. Test with `--dry-run` flag on `proxmox-backup.sh`.
